Background & Related Work

The binomial distribution is rooted in the concept of a Bernoulli trial, named after the Swiss mathematician Jacob Bernoulli, who discussed it in his work "Ars Conjectandi" (1713). A Bernoulli trial is a random experiment with exactly two possible outcomes, typically labeled "success" (with probability p) and "failure" (with probability 1-p). When a sequence of n independent Bernoulli trials is performed, the total number of successes observed follows a binomial distribution. This forms the basis for understanding discrete probability distributions where outcomes are binary.

Its mathematical formulation builds upon basic combinatorics, specifically combinations (denoted as "n choose k" or nCk), which calculate the number of ways to achieve k successes in n trials without regard to order. The binomial distribution is a cornerstone in introductory statistics and probability courses [1], providing a fundamental model for numerous real-world phenomena. Its relationship to other distributions, such as the Poisson distribution (as a limiting case for large n and small p) and the normal distribution (as an approximation for large n), highlights its central role in statistical theory [2].

Methodology

3.1 Theoretical Framework and Computational Setup

This report employs a theoretical approach, starting with the definition of a Bernoulli trial and the derivation of the binomial probability mass function (PMF). The PMF defines the probability of obtaining exactly k successes in n independent Bernoulli trials, each with a probability of success p, given by:
P(X = k) = C(n, k) * pk * (1-p)(n-k)
where C(n, k) = n! / (k! * (n-k)!) is the binomial coefficient, representing the number of ways to choose k successes from n trials.

To illustrate these concepts and perform calculations, Python 3.x with the `scipy.stats` library is used. This library provides functions for various probability distributions, including `binom.pmf` for calculating point probabilities and `binom.cdf` for cumulative probabilities. Visualization is achieved using `matplotlib.pyplot`.

binomial_calculator.py 
import numpy as np
from scipy.stats import binom
import matplotlib.pyplot as plt

# Define parameters for the binomial distribution
n = 10  # Number of trials
p = 0.5 # Probability of success on each trial

# Calculate probability of exactly k successes
k_successes = 5
prob_k_successes = binom.pmf(k_successes, n, p)
print(f"Probability of exactly {k_successes} successes in {n} trials with p={p}: {prob_k_successes:.4f}")

# Generate probabilities for all possible number of successes (0 to n)
x = np.arange(0, n + 1)
pmf_values = binom.pmf(x, n, p)

# Plotting the PMF
plt.figure(figsize=(8, 5))
plt.bar(x, pmf_values, color='skyblue', edgecolor='black')
plt.title(f'Binomial Distribution (n={n}, p={p})')
plt.xlabel('Number of Successes (k)')
plt.ylabel('Probability P(X=k)')
plt.xticks(x)
plt.grid(axis='y', linestyle='--', alpha=0.7)
plt.show()

Results

The application of the binomial distribution framework yields precise probabilities for discrete events with binary outcomes. For a binomial random variable X with parameters n (number of trials) and p (probability of success), the expected number of successes (mean) is E[X] = n*p, and the variance is Var(X) = n*p*(1-p). These properties provide quick insights into the distribution's characteristics without needing to calculate all probabilities.

Consider an example where we perform 10 coin flips (n=10) and the probability of heads is 0.5 (p=0.5). The expected number of heads is 10 * 0.5 = 5. The variance is 10 * 0.5 * 0.5 = 2.5. The probability mass function provides the likelihood for each possible number of heads (k=0 to 10).

Discussion

5.1 Interpretation of Results

The results demonstrate the power of the binomial distribution in quantifying uncertainty for repeated binary events. As observed in Table 1, for a fair coin (p=0.5), the probabilities peak at the expected value (k=5), indicating the most likely outcome is 5 heads in 10 flips. The probabilities decrease symmetrically as we move further from the mean. This symmetrical bell-like shape is characteristic of binomial distributions when p is close to 0.5. For scenarios where p deviates significantly from 0.5, the distribution would exhibit a clear skew, highlighting the dominant outcome. For example, if p=0.1, the probabilities would be much higher for k=0 or k=1, and rapidly diminish for larger k values. This predictive capability is invaluable in diverse fields such as quality control, where one can assess the probability of a certain number of defective items, or in marketing, for estimating the number of successful conversions from a campaign.

5.2 Limitations and Assumptions

While robust, the binomial distribution relies on several critical assumptions that, if violated, can lead to inaccurate predictions. These include:

  • **Fixed Number of Trials (n):** The number of repetitions of the experiment must be predetermined and constant.
  • **Independence of Trials:** The outcome of one trial must not influence the outcome of any other trial.
  • **Only Two Outcomes (Success/Failure):** Each trial must result in one of exactly two mutually exclusive outcomes.
  • **Constant Probability of Success (p):** The probability of success must remain the same for every trial.

Threats to validity arise if, for instance, the "probability of success" changes over time (e.g., a machine's defect rate worsening), or if trials are not independent (e.g., sampling without replacement from a small population). In such cases, alternative distributions like the hypergeometric distribution or more complex statistical models would be more appropriate. The sample size, `n`, must also be considered; for very large `n`, approximations like the normal distribution might be used, which have their own set of assumptions and limitations.

Appendix

This appendix includes an expanded Python script to generate the probability mass function (PMF) and cumulative distribution function (CDF) plots for binomial distributions with varying parameters, showcasing how changes in 'p' affect the distribution's shape and skewness.

extended_binomial_plots.py 
import numpy as np
from scipy.stats import binom
import matplotlib.pyplot as plt

def plot_binomial_distribution(n, p_values):
    plt.figure(figsize=(12, 6))
    for p in p_values:
        x = np.arange(0, n + 1)
        pmf_values = binom.pmf(x, n, p)
        plt.plot(x, pmf_values, marker='o', linestyle='-', label=f'p={p}')

    plt.title(f'Binomial Probability Mass Function for n={n}')
    plt.xlabel('Number of Successes (k)')
    plt.ylabel('Probability P(X=k)')
    plt.xticks(x)
    plt.grid(axis='y', linestyle='--', alpha=0.7)
    plt.legend()
    plt.show()

def plot_binomial_cdf(n, p_values):
    plt.figure(figsize=(12, 6))
    for p in p_values:
        x = np.arange(0, n + 1)
        cdf_values = binom.cdf(x, n, p)
        plt.step(x, cdf_values, where='post', label=f'p={p}')

    plt.title(f'Binomial Cumulative Distribution Function for n={n}')
    plt.xlabel('Number of Successes (k)')
    plt.ylabel('Cumulative Probability P(X≤k)')
    plt.xticks(x)
    plt.grid(axis='y', linestyle='--', alpha=0.7)
    plt.legend()
    plt.show()

# Example usage:
n_example = 20
p_examples = [0.1, 0.5, 0.9]

plot_binomial_distribution(n_example, p_examples)
plot_binomial_cdf(n_example, p_examples)
